1.3. Product Background
In recent years, the demand for watching movies in cinemas has increased significantly, especially for multi-branch cinema chains. Customers expect convenient services that allow them to easily browse movie information, check showtimes, select seats, and book tickets online.
However, managing movie schedules, ticket bookings, customer information, and revenue through manual processes or separate systems often leads to inefficiencies and errors. Customers may also experience difficulties when accessing movie information and booking tickets.
To address these challenges, the Cinema Management System is developed as a centralized platform for both online ticket booking and cinema operation management. The system enables customers to search for movies, view showtimes, and book tickets conveniently. At the same time, it helps staff, managers, and administrators efficiently manage movies, screening schedules, customers, bookings, and business performance.

1.4. Major Features

FE-01: Browse, search, and filter movies available in the cinema system.
FE-02: View detailed movie information and save movies to a personal favorites list.
FE-03: Create, update, and maintain movie records.
FE-04: Book movie tickets online for available showtimes.
FE-05: View booking history, cancel bookings, and access electronic tickets.
FE-06: Print/reissue tickets, support seat changes, and manage customer ticket bookings.
FE-07: View and manage movie showtimes.
FE-08: View and manage cinema rooms and room list
FE-09: Purchase and manage food and beverage products.
FE-10: View revenue reports and overall system statistics.

3.2.2 Manage Showtimes

ID and Name
	UC15 – Manage Showtimes 
	
Created by
	Trương Thế Anh
	
Primary Actor
	Branch Manager 
	Secondary Actor
	System
	
Description
	Branch Manager creates, updates, deletes, or views movie showtimes, including managing schedules, cinema rooms, and seat availability for each movie screening. 
	
Preconditions
	Branch Manager is logged into the system.
Movies and cinema rooms already exist in the system.
	
Main Sequence
	Branch Manager navigates to the "Showtime Management" section.
The system displays the current list of showtimes with details (Movie, Date, Time, Cinema Room, Status).
Branch Manager selects an action: Create Showtime, Update Showtime, Delete Showtime, or View Showtime Details.
Branch Manager enters or modifies showtime information:
Movie
Cinema Room
Date
Start Time
End Time
Status
Branch Manager clicks "Save/Confirm".
The system validates the input data.
The system saves the changes to the database.
The system displays the message:"Showtime information updated successfully."
	
Alternative Sequences
	6a. Invalid showtime information
The system displays the message:"Invalid input. Please check date, time, or cinema room."
The system highlights invalid fields for correction.
6b. Showtime conflicts with an existing schedule
The system displays the message:"The selected cinema room already has a scheduled showtime during this period."
Branch Manager must choose another room or time.
3a. Branch Manager deletes a showtime that already has bookings
The system displays the message:"Cannot delete this showtime because tickets have already been booked."
The system suggests changing the status to "Cancelled" instead of deleting.
	
Post conditions
	Showtime information is successfully added, updated, or removed.
Updated showtimes are immediately reflected on the customer booking system.
Cinema room schedules are updated accordingly.
Users can proceed to ticket booking if seats are available.
	
Business Rule
	BR-02, BR-10, BR-11, BR-14, BR-15 .

4. Non-Functional Requirements
4.1 External Interfaces
This section specifies how the cinema management system interacts with users, external hardware, and third-party software systems.
User Interfaces (UI):
The system must provide a responsive web interface optimized for both desktop browsers (Google Chrome, Microsoft Edge, Safari) and mobile viewports to ensure seamless navigation across all customer and administrative screens.
The interface design must strictly follow modern UI/UX guidelines, utilizing intuitive navigation patterns, clear error messages, and visual confirmation for transactional states (e.g., color-coded seating maps).
Hardware Interfaces:
Ticket Scanners: The system must interface with handheld or counter-top barcode/QR code scanners at the validation gates to instantly read and parse ticket tokens from the Ticket Validation screen.
Thermal Ticket Printers: The system must properly format and send raw print jobs to industrial thermal receipt printers at the counter for physical ticket issuance (Ticket Print screen).
Software & API Interfaces:
Payment Gateway API: Seamless integration with third-party payment gateways (e.g., VNPay, MoMo, or standard banking APIs) via secure Webhooks to synchronize transactional data from the Payment Gateway Interface.
Mailing Service API: Integration with automated SMTP or transactional email services (e.g., SendGrid, Mailgun) to instantly dispatch account activation codes, password reset OTPs, and dynamic electronic tickets.

4.2 Quality Attributes
4.2.1 Usability 
Learnability: The user interface for the web portal must be intuitive enough that a first-time customer can complete a full movie ticket and combo booking within 3 minutes without external guidance.
Accessibility: The layout must maintain a high contrast ratio for readable typography, and the real-time seat mapping components must use distinctive shapes or layout structures alongside colors to assist color-blind users.
Operational Ease: The staff and management dashboards must require minimal data entry, utilizing smart defaults, dropdown selectors, and automated calculations for F&B counter sales and ticket reissuance.
4.2.2 Reliability 
System Availability: The cinema management system must maintain an overall uptime of 99.9% during standard operating hours (6:00 AM to 2:00 AM daily), excluding scheduled late-night maintenance windows.
Fault Tolerance: In the event of a network disruption or payment gateway timeout during a transaction, the system must gracefully recover without losing session data or leaving transactional states corrupted.
Data Integrity & Recovery: The underlying database must perform automated incremental daily backups. In case of a critical hardware crash, the recovery time objective (RTO) must be under 2 hours, with a recovery point objective (RPO) limiting data loss to less than 1 hour.
4.2.3 Performance
Response Time: For standard navigational requests and data queries (such as rendering the Movie Details or filtering showtimes), the system must return data within 1.5 seconds under normal server loads.
Concurrent Transaction Handling: The booking and checkout processing services must be capable of handling at least 500 concurrent seat-locking and payment transactions simultaneously without performance degradation during peak box-office hours (e.g., blockbusters opening weeks).
Real-time Synchronization: The live seat layout updates on the View Real-time Room Layout screen must broadcast changes to all active client instances within 500 milliseconds of a seat status change on the database.

4.2.4 Security
Data Encryption: All sensitive communication, personal identity data, and monetary transactional payloads transmitted between client browsers and the backend servers must be encrypted using Transport Layer Security (TLS 1.3 / HTTPS). Passwords must be hashed using strong cryptographic algorithms (e.g., bcrypt) before database storage.
Access Control & Session Management: The system must rigorously enforce Role-Based Access Control (RBAC) as defined in the Screen Authorization matrix. Active sessions must automatically expire after 15 minutes of user inactivity, requiring re-authentication to prevent unauthorized access at unattended public or staff terminals.
Input Validation & Protection: Every input form across all screens (including registration, search queries, and administrative fields) must undergo strict server-side sanitization and structural validation to actively block malicious exploits such as Cross-Site Scripting (XSS) and SQL Injection attacks.
