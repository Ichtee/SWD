Package Descriptions
No
	Package
	Description
	
01
	FrontEnd 
	A client-side application built with React, responsible for rendering the user interface and handling user interactions. 
	
02
	FrontEnd :: routes 
	Configures routing and navigation between screens in the React app 
	
03
	FrontEnd :: pages 
	Contains Page Components that map directly to specific URL routes. 
	
04
	FrontEnd :: components 
	Contains Reusable Components designed independently to be shared across multiple pages. 
	
05
	FrontEnd :: services 
	Contains API fetching logic to send and receive data asynchronously with the Node.js server. . 
	
06
	FrontEnd :: utils 
	Contains shared helper/utility functions within the React app (e.g., date formatting, LocalStorage management, form validation).  
	
07
	BackEnd 
	A server-side application built on Node.js,  responsible for handling business logic and security. 
	
08
	BackEnd :: routes 
	Defines API Endpoints to receive HTTP requests sent from the React frontend. 
	
09
	BackEnd :: middlewares 
	Express intermediary functions that process incoming requests before they reach the controller
	
10
	BackEnd :: services 
	The core Business Logic layer. It drives the primary workflows of the application (e.g., validating ticket pricing, applying promo codes, processing concurrent seat locking, and calling third-party payment gateways like Stripe/MoMo). It remains independent of database syntax. 
	
11
	BackEnd :: models 
	Defines database schemas/entities and executes queries, typically utilizing an ORM or Query Builder 
	
12
	BackEnd :: config 
	Stores Node.js environment variables, port configurations, and establishes the database connection pool to MySQL. 
	
13
	BackEnd :: repositories 
	The Data Access layer. Acts as the sole abstraction layer between services and the database. It isolates database query logic. If the database engine or ORM changes in the future, only this package requires modifications. 
	
14
	BackEnd :: controllers 
	Receives requests from routes after passing through middlewares, extracts data (req.body, req.params), delegates execution to the services layer, and returns a JSON Response. 
	
15
	MySQL Database 
	A relational database management system that persistently stores all system and application data.
